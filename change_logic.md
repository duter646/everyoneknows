# 修改一些逻辑
你是一个资深软件工程师，需要修改当前项目的一些逻辑，并保证兼容和运行稳定。

## 答题系统赋分规则

### 0. 增加不同题量的试卷与计算时间功能
- 当前仅有20题，需要增加30/40/50题版本供用户选择，由此带来的评分问题下方解释。
- 时间计算按**停留在当前题目的总时间算**，从其他地方返回后继续累加。

### 1. 单题基础分

| 难度 | 单选题 (`single`) | 多选题 (`multiple`) |
|------|-------------------|---------------------|
| 易   | 10                | 15                  |
| 中   | 15                | 22.5                  |
| 难   | 20                | 30                  |

> 该基础分为单题满分基准（正确率 1.0 且时间因子 1.0 时的得分）。

### 2. 正确率系数

**单选题**  
- 用户选项与正确选项完全一致 → 正确率 `correctness = 1`  
- 否则 → `correctness = 0`（该题直接 0 分，不计算时间因子）

**多选题**  
- **包含任何错误选项（所选集合 ⊈ 正确答案集合）→ 正确率 = 0，直接 0 分**  
- 若所选均为正确选项（所选 ⊆ 正确答案），则按漏选比例计算：  
  \[
  \text{correctness} = \frac{|\text{用户选中的正确选项}|}{|\text{正确选项总数}|}
  \]
  例如：正确选项 {A,B,C}，用户选 {A,B} → 正确率 = 2/3 ≈ 0.667  
  用户选 {A} → 正确率 = 1/3 ≈ 0.333  
- 未选任何选项 → 正确率 = 0

### 3. 时间因子（含秒答惩罚）

**可配置常量**

| 参数 | 符号 | 默认值 | 说明 |
|------|------|--------|------|
| 激励强度 | α | 0.5 | 决定标准时间内加速得分及超时扣分的力度 |
| 保底/惩罚因子 | β | 0.5 | 超时最低得分倍率，同时也是秒答惩罚倍率 |
| 最小答题时间 | T_min | 2 秒 | 低于此值将直接施加惩罚 |
| 标准答题时间 | T | 易: 7s<br>中: 12s<br>难: 20s | **多选题标准时间需要*1.5** |

**计算规则**  
设实际答题耗时为 `t` 秒。

1. **秒答惩罚（t < T_min）**  
   若 `t < 3`，**时间因子 `f` 直接取 β（即 0.5）**，不再进行奖励计算。

2. **正常耗时（t ≥ T_min）**  
   - 计算时间比：`r = t / T`  
   - 若 `r ≤ 1`（标准时间内）：  
     `f = 1 + α × (1 - r)`  
   - 若 `r > 1`（超时）：  
     `f = max(β, 1 - α × (r - 1))`

**效果说明**  
- 标准时间内，越快得分越高，最高约 1 + α（若 t = T_min，r=0.15 时 f≈1.425）  
- 超时后分数线性下降，但不会低于 β（0.5）  
- 秒答（<2 秒）直接惩罚到 β，杜绝脚本刷分

### 4. 单题最终得分

\[
\text{单题得分} = \text{基础分} \times \text{正确率} \times \text{时间因子}
\]

- 正确率为 0 时直接返回 0 分，不计算时间因子。
- 得分保留一位小数。

### 5. 多题归一化方案（不同题目总数处理）

由于不同用户的试卷可能包含不同数量的题目，直接累加单题得分会造成不公平。因此采用**得分率映射**的方式，将原始总分归一化到百分制。

**定义**  
- **试卷基准满分** = 该份试卷所有题目基础分之和（即假定每道题正确率=1、时间因子=1 的总分）  
  \[
  \text{base\_total} = \sum_{i=1}^{n} \text{基础分}_i
  \]
- **用户实际总分** = 该用户在该试卷所有题目的单题得分之和  
  \[
  \text{user\_total} = \sum_{i=1}^{n} \text{单题得分}_i
  \]
- **归一化得分（百分制）**  
  \[
  \text{final\_score} = \frac{\text{user\_total}}{\text{base\_total}} \times 100
  \]
  结果保留一位小数。

**说明**  
- 归一化后，用户分数可横向对比，不再受题目数量影响。  
- 当用户答题又快又准时，可超过 100 分（体现速度奖励），例如全部秒答满分且时间奖励较高。  
- 如果需要限制最高分（如封顶 150 或 200），可在配置中增加 `MAX_SCORE` 常量，但默认不限制，由产品决定。

### 6. 完整计算示例

#### 示例试卷（共 3 题）
1. 单选题，难度易，正确选项 B  
2. 多选题，难度中，正确选项 {A, B, C}  
3. 多选题，难度难，正确选项 {A, C}

**试卷基准满分** = 10 (单选易) + 22 (多选中) + 30 (多选难) = **62**

#### 用户作答记录与得分

**第1题**：选 B，耗时 8 秒  
- 基础分 10，正确率 1.0  
- t=8 ≥ 3，T=20，r=0.4，f=1+0.5×(1-0.4)=1.3  
- 得分 = 10×1.0×1.3 = **13.0**

**第2题**：选 {A, B}（漏选，无错选），耗时 30 秒  
- 基础分 22，正确率 = 2/3 ≈ 0.667  
- t=30 ≥ 3，T=40，r=0.75，f=1+0.5×(1-0.75)=1.125  
- 得分 = 22×0.667×1.125 ≈ **16.5**

**第3题**：选 {A, D}（错选 D），耗时 5 秒  
- 存在错误选项 → 正确率 = 0  
- 得分 = **0**

**用户实际总分** = 13.0 + 16.5 + 0 = **29.5**

**归一化最终得分**  
\[
\frac{29.5}{62} \times 100 \approx 47.6\ \text{分}
\]

### 由于各个专业出现的频率不同，所以按照**用户对应得分/该专业出现的标准分之和（不算速度奖励最多能拿到几分）**计算，类似于得分率，注意分母为0的情况。由此得到各专业的得分率，然后按照与目前相同的归一化方法得出预测，考虑到可能不同专业概率差异过小，可以加一个类似于较弱的softmax函数。
